# Personal Secretary Bot 🐱

LINEを唯一のインターフェースにした**個人秘書システム**。
現在は **勤怠管理モジュール** を実装・運用中。フリーランスの自分の稼働を、面倒な打刻なしで正確に記録するために作った自分用ツール。

> 「いずれ秘書を雇うほどではないが、定型的な記録・通知は自動化したい」という個人事業主のリアルな課題を、LINEだけで完結する形で解決することを目指しています。勤怠管理はその最初のモジュールです。

## なぜ「秘書システム」として作ったか

単機能の勤怠ツールではなく、**最初から拡張を前提とした設計**にしています。

フリーランスとして働く中で自動化したい定型作業は勤怠だけではありません(タスク管理、メモ、リマインド、日次の振り返り等)。それらを毎回バラバラのツールやスクリプトで作るのではなく、**LINEという1つの窓口に集約し、機能をモジュールとして足していける基盤**にしておく方が、長期的に運用しやすいと考えました。

そのため、コマンドのルーティングをハンドラマップで管理し、**新機能の追加がルーティングのコアに影響しない構造**にしています(詳細は下記「拡張性のある設計」参照)。

## 現在の機能(Module 1: 勤怠管理)

- **LINEだけで打刻** — 専用アプリ不要、いつものLINEで
- **Notion自動保存** — 退勤時に日付・出退勤時刻・休憩合計・実質稼働時間を1行追加
- **ポモドーロ通知** — 作業開始から50分/60分でLINE通知。集中の区切りを促す
- **休憩の自動集計** — 休憩の累計を自動計算し、実質稼働時間に反映
- **一人運用前提の安全設計** — 自分のUser ID以外からのメッセージは無視

### 使い方

| コマンド | 動作 |
|---|---|
| `出勤` | 勤務開始。ポモドーロ通知のカウント開始 |
| `休憩` | 休憩開始。通知を一時停止 |
| `再開` | 休憩終了。休憩時間を加算してカウント再開 |
| `退勤` | 勤務終了。Notionに記録を保存 |
| `ヘルプ` | 利用可能なコマンド一覧を表示 |

## 拡張性のある設計

このシステムの中核は `COMMAND_HANDLERS`(コマンド → ハンドラ関数のマップ)です。

```js
const COMMAND_HANDLERS = {
  '出勤': handleClockIn,
  '休憩': handleBreakStart,
  '再開': handleBreakEnd,
  '退勤': handleClockOut,
  'ヘルプ': handleHelp,
};
```

Webhook受信処理(`doPost`)は、このマップを引いて対応するハンドラを呼ぶだけ。
つまり**新しい秘書機能を追加するときは、ハンドラ関数を実装してこのマップに1行足すだけ**で済み、ルーティングのコアには一切手を入れません。

```js
// 例: タスク管理モジュールを追加する場合
const COMMAND_HANDLERS = {
  ...(既存),
  'タスク追加': handleTaskAdd,   // ← 追加するのはこの2行だけ
  'タスク一覧': handleTaskList,  // ← doPost は変更不要
};
```

### 拡張ロードマップ(設計上の想定)

| モジュール | 想定コマンド | 状態 |
|---|---|---|
| Module 1: 勤怠管理 | 出勤 / 休憩 / 再開 / 退勤 | ✅ 実装・運用中 |
| Module 2: タスク管理 | タスク追加 / タスク一覧 / タスク完了 | 設計済み |
| Module 3: メモ | メモ(Notion別DBに保存) | 構想 |
| Module 4: リマインド | リマインド(時刻指定でpush) | 構想 |
| Module 5: 日次サマリー | サマリー(稼働+タスクを集約通知) | 構想 |

> Module 2以降は未実装です。ここで示しているのは「**この設計ならこう拡張できる**」という設計の妥当性であり、機能の約束ではありません。実装は必要になったタイミングで順次行います。

## 技術スタック

- **Google Apps Script** — サーバーレス実行環境(無料・トリガー機能)
- **LINE Messaging API** — Webhook受信 / reply・push送信
- **Notion API** — ログのデータベース保存
- **PropertiesService** — 状態管理と認証情報の保管

## セットアップ

### 1. 認証情報を準備

- **LINE**: [LINE Developers](https://developers.line.biz/) でMessaging APIチャネルを作成し、Channel access token を発行
- **Notion**: [Notion Integrations](https://www.notion.so/my-integrations) でインテグレーションを作成し、Internal Integration Secret を取得。対象データベースにインテグレーションを接続

### 2. Notionデータベースを用意

以下のプロパティを持つデータベースを作成:

| プロパティ名 | 種類 |
|---|---|
| 日付 | タイトル |
| 出勤 | 日付 |
| 退勤 | 日付 |
| 休憩合計(分) | 数値 |
| 実質稼働(時間) | 数値 |

### 3. GASにコードを配置

`Code.gs` の内容を Google Apps Script プロジェクトに貼り付け。

### 4. スクリプトプロパティを設定 ⚠️重要

GASエディタの **「プロジェクトの設定」→「スクリプト プロパティ」** から、以下の4つを登録する。**コードには認証情報を一切書かない。**

| プロパティ名 | 値 |
|---|---|
| `CHANNEL_ACCESS_TOKEN` | LINEのChannel access token |
| `MY_USER_ID` | 自分のLINE User ID |
| `NOTION_TOKEN` | NotionのIntegration Secret |
| `NOTION_DB_ID` | NotionデータベースのID |

設定後、`checkConfig` を実行して全て「設定済み」と出ればOK。

### 5. デプロイとWebhook設定

1. GASを「ウェブアプリ」としてデプロイ(アクセス: 全員)
2. デプロイURLを LINE Developers のWebhook URLに設定
3. `testPush` を実行してLINEに通知が届くか確認

### 6. ポモドーロ通知トリガーを設定

GASのトリガーで `notifyElapsed` を **1分おき** に実行するよう設定。

## セキュリティ方針

- 認証情報は**スクリプトプロパティのみ**で管理し、ソースコードには含めない
- `checkConfig` はトークンの中身を出力せず、設定有無のみ確認する
- 自分のUser ID以外からのメッセージは処理しない(一人運用前提)
- リポジトリには認証情報を一切コミットしない(`.gitignore` で `.clasp.json` 等を除外)

## 設計メモ

- **コマンドルーター**: `switch` ではなくハンドラマップを採用。機能追加時にコアロジックを変更しないための判断
- **状態管理**: 出退勤・休憩の状態を `PropertiesService` にJSONで保持。GASはステートレスなので外部に状態を持たせる必要がある
- **ポモドーロのリセット**: 休憩 → 再開のタイミングで通知フラグをリセットし、再開時点から再カウント
- **通知の冪等性**: `notified50` / `notified60` フラグで二重通知を防止。push成功時のみフラグを立てる(送信失敗時は次のトリガーで再送)
- **Notion保存の分離**: 保存処理をモジュールごとに独立させ、他モジュール追加時に勤怠ロジックと干渉しない設計
